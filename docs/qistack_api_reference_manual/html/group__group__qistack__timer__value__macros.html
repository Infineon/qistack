◆ CY_QI_TIMER_PKT_TIMEOUT_TIME

#define CY_QI_TIMER_PKT_TIMEOUT_TIME   (450u)

Packet timeout for ASK path switch in mS.

This number ensures that we switch ASK decode mode if no packet was received within this timeout. Since there is a possibility of a wrong switch, we need to ensure that we come back to original mode within CEP packet timeout and be able to receive a packet. To allow for this, we use 1800ms / 4 = 450ms.
